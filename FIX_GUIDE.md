# 超级管理员账号重复登录问题修复指南

## 问题描述
原问题：超级管理员账号 15900000000，别人用其他电脑登录时会重新创建一个相同手机号的账号。

## 已完成的修复

### 1. 修改后端登录逻辑 (server/routes/user.js)
- 优先级调整：手机号登录优先级最高
- 如果传入手机号，优先按手机号查询用户
- 如果已存在该手机号用户，会将新的 openid 绑定到现有账号上
- 这样即使用不同的微信 openid，只要手机号相同，就能登录到同一个账号

### 2. 添加数据库唯一约束
- 给 users 表的 phone 字段添加唯一约束 `uk_phone`
- 给 users 表的 openid 字段添加唯一约束 `uk_openid`
- 从数据库层面防止重复创建相同手机号的账号

### 3. 创建的工具脚本
- `database/migrate_add_unique_constraints.js` - 给现有数据库添加唯一约束的脚本
- `database/cleanup_duplicate_admin.js` - 清理已存在的重复账号脚本

## 修复步骤

### 步骤 1：运行迁移脚本添加唯一约束
```bash
cd e:\my-vue3-project
node database/migrate_add_unique_constraints.js
```

### 步骤 2：如果存在重复账号，运行清理脚本
```bash
node database/cleanup_duplicate_admin.js
```

### 步骤 3：重启服务器
确保服务器使用更新后的代码运行。

## 修复后的工作流程

现在，当用户使用微信登录并获取到手机号时：
1. 后端优先按手机号查询
2. 如果找到该手机号的用户（如超级管理员 15900000000）
3. 将用户的 openid 绑定到该账号（如果该账号还没有 openid）
4. 返回该账号的登录凭证
5. 无论用哪个设备登录，只要手机号相同，都能登录到同一个账号

## 注意事项

- 如果数据库中已存在重复的手机号，运行迁移脚本时会失败
- 需要先运行清理脚本，或者手动清理重复数据
- 清理脚本会保留 ID 最小的那个账号，其他重复账号会被标记
